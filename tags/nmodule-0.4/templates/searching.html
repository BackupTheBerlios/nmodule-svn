NModule Documentation

Search Paths

This document describes how NModule uses search paths to find modules.

Introduction/Explanation

Your application will probably want a way to space out modules in a way seperate from the main application. NModule provides a mechanism for this similar to PATH to achieve this. Internally, this is just a list of directories to search for modules, externally, it can be in any form you want. NModule just uses your search path. You must add at least one directory to the search path before you can load a module, this is done by doing something like: 

controller.SearchPath.Add ("path/to/modules");

NModule adds the paths to the AppDomain's search path when loading modules, so there are some rules to the directories. Beyond the usual requirements (you must have the permission to read the files and directories), the directories must be subdirectories of the directory your application resides in. For example, if your application resides in a directory called foo, your modules can live in foo/bar, or foo/bar/baz, but not anything that is not underneath foo.
