NModule Documentation

Entry/Exit Handlers

This document describes the function of entry and exit handlers in NModule.

Introduction

NModule provides mechanisms for modules to add/remove roles from the controller in order to add new functionality to the application. This behavior is done using entry/exit handlers, which may also be used for initialization and deinitialization of module-specific data structures.

This behavior is optional, and modules are not required to provide a type which implements the IModule interface. The behavior is provided to facilitate functionality which we believe belongs in NModule.

IModule

In order to make use of these facilities, the module needs to define a type which implements the IModule interface. This interface consists of two methods, an entry handler and an exit handler. Both handlers take one argument, which is an object variable that houses the controller that loaded the module. It is always of type ModuleController, and is only sent as an object to avoid circular dependencies between NModule.Core and NModule.Core.Loader.

Entry Handlers

Entry handlers are called when the module is loaded. Modules can use this to register new roles with the controller, or initialize needed data structures.

Exit Handlers

Exit handlers are called when the module is unloaded. This can be used to unregister roles the module registered with the controller, or clean up after itself.
